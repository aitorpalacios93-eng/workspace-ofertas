# 00 Principios

## Objetivo

Operar un solo sistema interno capaz de servir tres rutas comerciales distintas sin romper posicionamiento, pricing ni mensaje.

## Principios obligatorios

1. Se vende un resultado, no una lista de servicios.
2. Cada prospecto entra en una sola ruta comercial.
3. Cada propuesta recomienda una oferta core y, como maximo, dos add-ons.
4. El backend de automatizacion no es oferta de entrada.
5. El sistema puede decir "no" a una propuesta completa cuando el fit es bajo.
6. La evidencia manda sobre la intuicion. Si faltan pruebas, se marca como hipotesis.
7. `US_HISPANIC` existe como playbook de alto upside, pero no domina la operacion hasta tener caso ES documentado.
8. El orden correcto es venta, caso, refinado, automatizacion.
9. NotebookLM se usa para reforzar contexto, no para inventar problemas ni justificar venta forzada.

## Lo que nunca debe pasar

- Mostrar pricing de entrada local a una PYME B2B seria.
- Mezclar lenguaje de "video", "web" y "ads" como si fueran la oferta.
- Proponer automatizacion pesada a un prospecto que aun no confia.
- Generar PDFs bonitos para prospectos con bajo fit.
- Activar outreach fuerte en USA sin prueba social minima.

## Regla de posicionamiento

La marca es una sola. Lo que cambia es el wrapper comercial:

- `ES_LOCAL`: activacion y claridad.
- `ES_B2B`: credibilidad comercial y sistema.
- `US_HISPANIC`: trust, conversion y follow-up.

## Artefactos minimos por oportunidad

1. Diagnostico.
2. Fit score.
3. Recomendacion de ruta.
4. Oferta recomendada.
5. Pack de cierre.
6. Registro posterior de objeciones y outcome.
