# 04 Orden de Ejecucion

## Problema que corrige este documento

Una arquitectura buena puede fracasar si se ejecuta como proyecto de software antes de validarse como proyecto comercial.

## Orden correcto

1. Vender.
2. Documentar.
3. Refinar el playbook.
4. Automatizar lo repetido.

## Prioridad operativa 60 dias

- `ES_B2B`: 50%
- `ES_LOCAL`: 40%
- `US_HISPANIC`: 10%

## Fase 1

Objetivo: usar conversaciones reales, no teoria.

- Priorizar cualquier propuesta B2B ya viva.
- Buscar un cierre rapido en `ES_LOCAL`.
- En `US_HISPANIC`, preparar ICP, mensajes y catalogo, pero no hacer outreach masivo.

## Fase 2

Objetivo: convertir experiencia real en sistema.

- Registrar objeciones.
- Registrar pricing aceptado o rechazado.
- Registrar que promesa resonou.
- Ajustar routing y oferta con datos, no con ideas.

## Fase 3

Objetivo: automatizar solo lo que ya se repite.

- Implementar scripts donde haya patron estable.
- Mejorar plantillas segun cierres y perdidas.
- Activar `US_HISPANIC` de forma mas agresiva solo si ya existe un caso ES documentado.

## Gate de expansion USA

`US_HISPANIC` pasa de playbook a foco comercial real cuando se cumple al menos esto:

- un caso ES documentado
- una narrativa clara de resultado
- una oferta core afinada
- un proceso de propuesta y follow-up ya probado

## Recordatorio clave

No hace falta un sistema perfecto para vender. Hace falta un sistema suficiente para no perder claridad ni repetir errores.
